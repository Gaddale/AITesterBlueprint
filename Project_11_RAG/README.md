# Project 11 — Basic RAG System with ChromaDB, Nomic Embed & Groq

A fully local **Retrieval-Augmented Generation (RAG)** pipeline built over the VWO Login Dashboard Product Requirements Document. The system ingests a PDF, chunks and embeds it with **Nomic Embed Text v1**, stores vectors in a local **ChromaDB** instance, and answers natural-language questions using the **Groq LLM API** — all visualised in a single-page HTML explorer.

---

## Architecture

```
INGESTION PIPELINE
PDF ──► Chunking (1000 chars / 150 overlap) ──► Nomic Embed v1 (768-dim) ──► ChromaDB (local)

QUERY PIPELINE
User Question ──► Nomic Embed (search_query prefix) ──► Cosine Similarity Search
    └──► Top-K Chunks ──► Groq llama-3.1-8b-instant ──► Grounded Answer
```

| Backend | Flask + Flask-CORS |

## Setup & Run

### 1. Install dependencies

```bash
cd Project_11_RAG/Basic_RAG_Explain
pip install -r requirements.txt
```

> **Note:** First run downloads the Nomic Embed model (~548 MB) from HuggingFace.

### 2. Add your Groq API key

Create a `.env` file:

```env
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.1-8b-instant
```

Get a free Groq API key at [console.groq.com](https://console.groq.com).

### 3. Ingest the PDF

```bash
python3 ingest.py
```

This will:
- Extract text from all PDF pages
- Chunk into ~1000-character segments with 150-character overlap
- Generate Nomic embeddings with `search_document:` prefix
- Store everything in a local ChromaDB collection (`vwo_docs`)
- Save chunk metadata to `chunks.json` for the UI

Expected output:
```
INGESTION COMPLETE
Total chunks : 15
ChromaDB path: chroma_db/
Embed model  : nomic-ai/nomic-embed-text-v1
```

### 4. Start the Flask server

```bash
python3 app.py
```

Open **http://localhost:5011** in your browser.

## What the UI Shows

### Pipeline Diagram (always visible)
Two animated rows showing the full architecture:
- **Ingestion row** — PDF → Chunking → Nomic Embed → ChromaDB (always green = done)
- **Query row** — animates step-by-step as each query is processed

### Chunk Explorer (left panel)
- All 15 chunks displayed as cards with chunk ID, page number, word count, character count
- Searchable/filterable
- After a query, retrieved chunks glow with rank badges (🥇 🥈 🥉) and cosine similarity bars
- Click any card to expand the full text

### Query Interface (right panel)
- Free-text question input (Ctrl+Enter to submit)
- Top-K selector (1–5 chunks)
- Step-by-step loading indicators while embedding, retrieving, and generating
- Answer card with full LLM response and timing breakdown
- Retrieval flow diagram showing exactly which chunks fed the LLM

## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `GET /` | GET | Serves the HTML UI |
| `GET /api/chunks` | GET | Returns all chunks as JSON |
| `GET /api/stats` | GET | ChromaDB stats (count, collection, models) |
| `POST /api/query` | POST | Runs a RAG query, returns answer + retrieved chunks + timing |

### Example query request

```bash
curl -X POST http://localhost:5011/api/query \
  -H "Content-Type: application/json" \
  -d '{"query": "What are the login requirements for VWO?", "top_k": 3}'
```

### Example response (abbreviated)

```json
{
  "answer": "The login requirements include...",
  "retrieved_chunks": [
    { "id": "p1_c1", "page": 1, "rank": 1, "similarity": 0.77, "text": "..." },
    ...
  ],
  "timing": { "embed_ms": 141, "retrieve_ms": 2, "llm_ms": 370, "total_ms": 513 },
  "models": { "embed": "nomic-ai/nomic-embed-text-v1", "llm": "llama-3.1-8b-instant" }
}
```

## Chunking Strategy

| Parameter | Value |
|-----------|-------|
| Chunk size | 1000 characters |
| Overlap | 150 characters |
| Break preference | Sentence boundary (`.` or newline) within last 50% of chunk |
| Minimum chunk length | 50 characters (short fragments discarded) |
| Embedding prefix (docs) | `search_document: ` |
| Embedding prefix (queries) | `search_query: ` |

## Key Concepts Demonstrated

- **RAG Pipeline** — how to ground an LLM response in retrieved document context
- **Local Vector Store** — ChromaDB persistent storage with cosine similarity index
- **Open-Source Embeddings** — Nomic Embed v1 without any paid embedding API
- **Chunk Strategy** — character-based chunking with sentence-aware boundary detection
- **Query–Document Asymmetry** — different prefixes for documents vs. queries (Nomic requirement)
- **Visual RAG Debugging** — see exactly which chunks influence each answer
